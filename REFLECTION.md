## How Microsoft Copilot Assisted in Development

Microsoft Copilot was used throughout the development of **InventoryHub** to support full-stack integration, debugging, JSON structuring, and performance optimization. It helped generate the initial Blazor WebAssembly `HttpClient` integration with the .NET Minimal API, including async API calls and reusable methods for retrieving product data from `/api/productlist`.

Copilot also assisted in resolving key issues such as incorrect API routes, CORS errors, and JSON deserialization problems. Its suggestions helped improve error handling, align front-end models with back-end JSON responses, and introduce a nested category structure for cleaner and more scalable data design.

For performance improvements, Copilot supported the implementation of caching on both the front end and back end. This reduced unnecessary API calls, improved response time, and made the application more efficient. Overall, Copilot helped accelerate development, improve code quality, and reinforce best practices while still requiring careful review, testing, and refinement.
